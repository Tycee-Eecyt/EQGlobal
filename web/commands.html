Mob Windows Admin Command Guide

Discord-Style Command Guide

The bulk ToD parser supports both !command and plain command text from copied Discord logs. Commands below are the current parser behavior in EQGlobal.

Write Commands

Command Purpose
!tod <mob> [time] Set ToD for a mob. If no time is provided, parser uses message timestamp.
!tod quake [time] / !quake [time] Apply one quake timestamp to all tracked mobs.
!todremove <mob> Clear ToD for a single mob.
!skip <mob> [time] Record skipped spawn count (+1).
!unskip <mob> Remove one skip from mob.
!alias ... Session alias mapping in parsed paste for name matching.

Read / Emulated Commands

Command Behavior
!todhistory <mob> Shown in preview feedback as history output.
!leaderboard Recognized and emulated in preview summary.
!timers, !schedule, !help Recognized as bot/read commands and ignored for writes.

Time Parsing Examples

  • !tod vindi now
  • !tod kdt 2026-02-17 14:09:00
  • !tod narandi [Mon Feb 16 23:46:58 2026]
  • !tod statue 15 minutes ago
  • !skip lodi 20 mins ago
  • !quake now

Accepted formats include ISO, US date/time, bracketed EQ timestamps, and relative values such as 2h ago, 15 minutes ago, or -30m.
